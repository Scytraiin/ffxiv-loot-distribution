# Loot History v1.0.4-alpha

## Summary
Structured loot-roll reintroduction release with improved item handling for unresolved loot and a cleaner full-mode browser layout.

## Highlights
- Reintroduced roll capture using a structured chat/log pipeline with pending-session correlation instead of the older text-only approach.
- Added roll details to expanded loot rows and item tooltips, along with a small row badge when a loot entry has attached rolls.
- Split the full browser toolbar into two grouped rows so the main window is easier to fit and scan at narrower widths.
- Reworked the expanded detail drawer into clearer sections for recipient, item metadata, context, rolls, and the original raw line.
- Switched favorites and blacklist behavior to stable item keys, allowing unresolved items and currencies to be pinned or hidden even without a resolved icon or item ID.

## Notes
- Roll tracking now prefers structured log and payload data, with text parsing kept only as a fallback.
- Existing ID-based favorites and blacklist entries are migrated forward automatically.
- As an alpha release, the main remaining thing worth validating in-game is whether all Need/Greed/Pass message variants expose the expected payload or parameter shapes.
